## 一、项目概述

### 1.1 项目简介

全双工语音对话系统是一个支持实时语音交互的智能对话系统，具备以下核心能力：

- **流式语音识别**：使用 Qwen ASR 17B 进行实时语音转文字
- **流式语义判断**：边说边判断用户是否说完
- **并行情绪识别**：实时分析用户情绪
- **全双工交互**：支持用户随时打断AI回复
- **多轮对话**：支持上下文连贯的多轮对话
- **工具调用**：支持天气查询、时间查询、设备控制等工具

### 1.2 技术栈

| 类别 | 技术选型 |
|------|----------|
| 编程语言 | Python 3.8+ |
| Web框架 | FastAPI + Uvicorn |
| 实时通信 | WebSocket |
| ASR模型 | Qwen ASR 17B (Paraformer) |
| 语义VAD | Qwen3-Omni-Flash |
| 情绪识别 | Qwen3-Omni-Flash |
| LLM | Qwen-Plus |
| TTS | Edge TTS |
| 声学VAD | WebRTC VAD |

#### 2.3.2 回调处理

```python
class StreamingASRCallback(RecognitionCallback):
    """
    流式ASR回调处理器

    使用 call_soon_threadsafe 在主线程事件循环中调度回调
    解决跨线程事件循环问题
    """

    def on_event(self, result):
        """收到识别结果 - 线程安全回调"""
        if self._on_result and self._loop:
            self._loop.call_soon_threadsafe(
                lambda: asyncio.create_task(
                    self._on_result(text, is_final=False)
                )
            )
```

class VoiceDialogSystem:
    """
    全双工语音对话系统 v3.0

    核心流程：
    1. 声学VAD检测语音起止
    2. 检测到人声 → 启动ASR + 情绪识别（并行）
    3. ASR流式输出 → 语义VAD流式判断
    4. 语义完整 → 文本+情绪 → LLM处理
    5. 打断 → 停止TTS → 新对话轮

// PCM编码
function floatTo16BitPCM(float32Array) {
    const int16Array = new Int16Array(float32Array.length);
    for (let i = 0; i < float32Array.length; i++) {
        const s = Math.max(-1, Math.min(1, float32Array[i]));
        int16Array[i] = s < 0 ? s * 0x8000 : s * 0x7FFF;
    }
    return int16Array;
}

## 八、性能优化

### 8.1 延迟优化

| 优化点 | 措施 | 效果 |
|--------|------|------|
| ASR延迟 | 流式处理 | 实时输出 |
| 语义判断 | 边说边判断 | 减少等待 |
| 并行处理 | asyncio.gather | 降低30-50% |
| 线程安全 | call_soon_threadsafe | 回调无阻塞 |

### 8.2 关键参数

| 参数 | 值 | 说明 |
|------|-----|------|
| ASR帧长 | 20ms | 平衡延迟与准确率 |
| 声学VAD阈值 | 400ms | <500ms要求 |
| 最大静音等待 | 2000ms | 超时保护 |
| 最小语音时长 | 300ms | 过滤短噪音 |
