## 一、系统概述

### 1.1 设计目标

设计一个**全双工语音对话系统 v3.0**，具备以下核心能力：

1. **实时语音采集与处理**：从麦克风采集音频流
2. **声学VAD**：使用WebRTC检测用户语音的起止与打断，阈值<500ms
3. **Qwen ASR 17B**：使用Qwen ASR 17B模型将语音转换为文本，帧长20ms
4. **Qwen Omni语义VAD**：使用Qwen3 Omni flash模型流式判断语义完整性
5. **并行情绪识别**：使用Qwen3 Omni flash模型识别用户情绪
6. **融合阶段**：文本和情绪同时输入LLM进行任务规划
7. **意图理解与任务规划**：LLM理解用户意图并规划任务
8. **工具调用**：支持SKILLS和MCP协议的工具调用
9. **结果总结**：LLM对工具调用结果进行总结
10. **语音合成**：通过Edge TTS将文本转换为语音输出
11. **全双工交互**：支持用户在系统说话时随时打断
12. **多轮对话**：支持上下文，对话连贯

声学VAD 和语义VAD的分工和定位：
# 1. 先一句话分清两者
- **声学 VAD（aVAD）**：听**声音有没有人在说话**
  如：paraformer-realtime-v2 里自带的 VAD、WebRTC VAD、阿里SenseVoice VAD
- **语义 VAD（sVAD）**：听**话有没有说完**（语义终点）
  如：Qwen3-Omni、Claude、GPT-RealTime 里的“端到端断句”

**全双工 = 边听边说 + 可打断 + 低延迟**
必须 **aVAD + sVAD 一起用**，缺一不可。

---

# 2. 全双工系统里的标准流程（最实用）
## 步骤1：声学 VAD 负责「抓起点」
- 检测到 **人声开始** → 打开音频流，送给 ASR
- 检测到 **静音** → 不立刻断，而是**交给语义VAD判断**

作用：
- 降噪
- 过滤非人声
- 不给ASR送无效音频

## 步骤2：语义 VAD 负责「抓终点」
ASR 实时出文字，sVAD 同时判断：说完、打断、拒识

满足：
**声学静音 ≥ 阈值（如 300~700ms） + 语义完整**
→ 判断为 **用户说完了**

## 步骤3：全双工核心逻辑（打断机制）
只要 aVAD 检测到 **用户重新说话**
→ **立刻打断 TTS / 模型回复**
→ 重新开始新一轮 ASR + sVAD

这就是**真·全双工**。

---

# 3. 两者在系统里的**精确分工**
## 声学 VAD 做：
- 人声起点检测
- 静音检测
- 噪声过滤
- 控制音频流开关
- **触发打断**

## 语义 VAD 做：
- 判断用户**是否说完意图**
- 避免把“思考停顿”当成结束
- 避免把“长句没说完”误触发
- 决定**什么时候让AI开始回复**

---

# 4. 超实用参数（直接拿去用）
- 声学 VAD 静音阈值：**300–700ms**
- 语义 VAD 最小响应延迟：**≥600ms**
- 打断延迟：**<200ms**
- 全双工目标端到端延迟：**<800ms**

---

# 5. 最终总结（最精炼）
- **声学 VAD = 管声音、管打断、管流**
- **语义 VAD = 管意图、管结束、管回复时机**
- **全双工 = aVAD 快 + sVAD 准 + 可打断**

---


### 1.2 系统特性

| 特性 | 说明 |
|------|------|
| 全双工 | 支持打断、实时响应 |
| 多模态输入 | 语音 + 文本 |
| 多模态输出 | 语音 + 文本 |
| 流式处理 | ASR流式输出 + 语义VAD流式判断 |
| 并行处理 | 语义VAD + 情绪识别并行 |
| 情绪融合 | 文本+情绪同时输入LLM |
| 工具调用 | 天气、提醒、音乐、设备控制 |
| 可配置 | 模型配置抽离为单独YAML文件 |

### 1.3 核心变化（v2.0 → v3.0）

| 项目 | v2.0 | v3.0 |
|------|------|------|
| ASR模型 | Qwen Omni统一处理 | **Qwen ASR 17B（独立）** |
| 语义VAD | 语音段结束后判断 | **流式文本实时判断** |
| 情绪识别 | Qwen Omni统一处理 | **Qwen Omni Flash并行处理** |
| 声学VAD阈值 | 固定300ms | **<500ms，交给语义VAD决策** |
| ASR帧长 | 30ms | **20ms** |

## 五、流式处理设计

### 5.1 核心流程

```python
async def process_audio_stream(self, audio_stream):
    """处理音频流 - 流式架构"""

    # 1. 声学VAD检测
    async for audio_chunk in audio_stream:
        is_speech = self.acoustic_vad.detect(audio_chunk)

        if is_speech:
            # 检测到人声开始 -> 打开音频流送给ASR
            if not self._stream_opened:
                self._stream_opened = True
                self._start_asr_stream()

            # 并行处理：ASR + 情绪识别
            await self._process_parallel(audio_chunk)
        else:
            # 检测到静音
            silence_duration = self.acoustic_vad.get_silence_duration()

            if silence_duration < 500:  # 阈值<500ms
                # 不立刻断，继续等待
                continue
            else:
                # 交给语义VAD判断
                if self.semantic_vad.is_complete():
                    # 语义完整 -> 进入融合阶段
                    await self._finalize_and_fuse()
                else:
                    # 语义不完整 -> 继续等待
                    continue
```

### 5.2 ASR流式输出 + 语义VAD流式判断

```python
async def _process_parallel(self, audio_chunk: bytes):
    """并行处理音频块"""

    # 并行执行：ASR + 情绪识别
    asr_task = asyncio.create_task(self.asr.process_chunk(audio_chunk))
    emotion_task = asyncio.create_task(self.emotion_recognizer.process_chunk(audio_chunk))

    # 等待ASR结果
    asr_text = await asr_task

    # ASR流式文本 -> 语义VAD流式判断
    if asr_text:
        semantic_result = await self.semantic_vad.judge(asr_text)

        if semantic_result.state == SemanticState.COMPLETE:
            # 语义完整，准备进入融合阶段
            self._pending_finalize = True

    # 情绪识别结果（并行进行）
    emotion_result = await emotion_task
    self._emotion_buffer = emotion_result
```

### 5.3 并行优势

| 优势 | 说明 |
|------|------|
| 降低延迟 | ASR流式输出，语义VAD边接收边判断 |
| 实时响应 | 情绪识别与语义VAD并行 |
| 智能判断 | 声学静音交给语义VAD决策，避免误断 |
| 更好体验 | 用户说完即响应，无需等待长时间静音 |

## 七、打断与多轮对话设计

### 7.1 打断机制

支持用户在AI说话时随时打断：

```
声学VAD检测到人声
    ↓
判断是否在SPEAKING状态
    ↓
是 → 检测到打断
    ↓
停止TTS播放
    ↓
开始新一轮对话
    ↓
保持上下文连续
```

### 7.2 实现细节

```python
async def _check_interrupt(self, audio_chunk: bytes) -> bool:
    """检查是否需要打断"""
    if not self.dialog_state.can_interrupt():
        return False

    # 声学VAD检测当前帧是否有语音
    is_speech = self.acoustic_vad.detect(audio_chunk)

    if is_speech:
        return True
    return False

async def _handle_interrupt(self) -> DialogResult:
    """处理打断"""
    # 停止TTS播放
    self.streaming_tts.stop()

    # 开始新一轮对话
    await self.dialog_state.force_state(DialogState.LISTENING, "用户打断")

    # 保持上下文（对话历史不清空）
    return DialogResult(
        text="",
        semantic_state=SemanticState.INTERRUPTED,
        is_interrupt=True,
        dialog_state=DialogState.LISTENING
    )
```

### 7.3 多轮对话上下文

```python
class ConversationManager:
    """多轮对话管理器"""

    def __init__(self, max_history: int = 10):
        self.history: List[Message] = []
        self.max_history = max_history

    def add_message(self, role: str, content: str, emotion: EmotionType = None):
        """添加消息到历史"""
        self.history.append(Message(
            role=role,
            content=content,
            emotion=emotion
        ))

        # 限制历史长度
        if len(self.history) > self.max_history * 2:
            self.history = self.history[-self.max_history * 2:]

    def get_context(self) -> List[Dict]:
        """获取对话上下文"""
        return [msg.to_openai_format() for msg in self.history]
```

额外说明
### 二、核心并行设计逻辑
1. **音频分流无延迟**：麦克风采集的音频帧通过内存队列分流，同时送给 VAD、ASR、情绪识别模块，无额外拷贝延迟；
2. **情绪识别轻量化**：用阿里 Qwen3-Omni-Flash模型，可和 ASR 并行完成；
3. **结果同步不阻塞**：语义VAD判断时，情绪结果如果已提前就绪，只需合并判断，不增加主流程延迟；
4. **打断逻辑优先**：声学VAD的打断信号可直接终止所有并行模块（ASR/情绪识别），保证全双工核心体验。

### 三、关键优化点（兼顾时延+并行）
- 把情绪作为LLM模型的输入参数，让回复适配情绪（如用户生气时，回复更耐心）；
- TTS层面调整语速/语调：开心时语速快、音调高，生气时音量大、语速稍快，悲伤时语速慢；
- 情绪结果缓存：短时间内（如3s）复用同一情绪结果，避免频繁切换导致回复风格混乱。
