## 2. 架构概述

### 2.1 架构目标

- **高性能**：首字响应 < 1s，工具调用 < 3s
- **高可用**：系统可用性 > 99.9%
- **可扩展**：支持水平扩展，工具插件化
- **易维护**：模块解耦，独立部署升级

### 2.2 架构原则

1. **分层架构**：接入层、业务层、模型层、数据层分离
2. **异步优先**：全链路异步，最大化并发能力
3. **流式输出**：SSE 流式响应，提升用户体验
4. **故障隔离**：服务熔断降级，避免级联故障

## 4. 核心组件设计

### 4.1 Thinker-Talker 双模型协作架构

```
用户输入
    │
    ▼
┌─────────────────────────────────────────────────────────┐
│                    意图路由器 (Intent Router)            │
│  - 简单问答 → Talker Only                               │
│  - 工具调用 → Talker + Thinker 并行                     │
│  - 复杂推理 → Thinker 主导                              │
└─────────────────────────────────────────────────────────┘
    │
    ├─────────────────────────────────────┐
    ▼                                     ▼
┌─────────────────┐               ┌─────────────────┐
│     Talker      │               │     Thinker     │
│  ┌───────────┐  │               │  ┌───────────┐  │
│  │ 快速响应   │  │  ◄─ 融合 ─►  │  │ 深度推理   │  │
│  │ 流式输出   │  │               │  │ 工具调用   │  │
│  └───────────┘  │               │  └───────────┘  │
│                 │               │        │        │
│  输出: 0.5-1s   │               │        ▼        │
└────────┬────────┘               │  ┌───────────┐  │
         │                        │  │ 工具执行   │  │
         │                        │  └───────────┘  │
         │                        │        │        │
         │                        │  输出: 2-3s     │
         │                        └────────┬────────┘
         │                                 │
         └─────────────┬───────────────────┘
                       ▼
              ┌─────────────────┐
              │   响应融合器     │
              │   (Merger)      │
              └────────┬────────┘
                       ▼
              ┌─────────────────┐
              │   流式输出      │
              │   (SSE Stream)  │
              └─────────────────┘
```

### 4.2 响应融合策略

| 场景 | Talker 行为 | Thinker 行为 | 融合方式 |
|------|-------------|--------------|----------|
| 简单问答 | 直接回答 | 不参与 | Talker 独立输出 |
| 天气查询 | "让我查一下..." | 调用天气 API | Thinker 结果追加 |
| 复杂规划 | 开场白 | 深度规划 | 分段融合输出 |
| 多模态 | 初步描述 | 图像分析+推理 | 渐进式补充 |

## 5. 模型选型方案

### 5.1 推荐配置

| 角色 | 推荐模型 | 备选模型 | 说明 |
|------|----------|----------|------|
| Talker | Qwen2-7B-Chat | GLM-4-9B-Chat | 轻量快速，对话能力强 |
| Thinker | Qwen2-72B-Chat | DeepSeek-V2 | 强推理，支持 Function Calling |
| Vision | Qwen2-VL-7B | InternVL2-8B | 多模态理解 |
| Embedding | BGE-M3 | M3E-large | 中文向量编码 |

### 5.2 硬件需求（私有化部署）

| 组件 | GPU 需求 | 内存需求 | 实例数 |
|------|----------|----------|--------|
| Talker (7B) | 1× A100 40G | 64GB | 2+ |
| Thinker (72B) | 4× A100 80G | 256GB | 1+ |
| Vision (7B) | 1× A100 40G | 64GB | 1+ |
| Embedding | CPU 即可 | 16GB | 2+ |

### 11.3 意图冲突检测模块

```
┌─────────────────────────────────────────────────────────────────────────┐
│                      Intent Conflict Detector                            │
│                                                                         │
│  输入：                                                                  │
│  - current_intent: 当前执行任务的意图                                    │
│  - new_intent: 新消息的意图                                              │
│  - context: 对话上下文                                                   │
│                                                                         │
│  检测规则：                                                              │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │ 1. 语义相似度检测                                                │   │
│  │    - 相似度 > 0.8: 补充信息                                      │   │
│  │    - 相似度 0.3-0.8: 部分相关                                    │   │
│  │    - 相似度 < 0.3: 完全无关                                      │   │
│  ├─────────────────────────────────────────────────────────────────┤   │
│  │ 2. 关键词检测                                                    │   │
│  │    - 取消词: "算了", "不用了", "取消"                             │   │
│  │    - 切换词: "换一个", "另外", "还有"                             │   │
│  │    - 补充词: "还要", "加上", "另外还"                             │   │
│  ├─────────────────────────────────────────────────────────────────┤   │
│  │ 3. 实体重叠检测                                                  │   │
│  │    - 相同实体类型: 可能是补充或修改                               │   │
│  │    - 不同实体类型: 可能是意图切换                                 │   │
│  └─────────────────────────────────────────────────────────────────┘   │
│                                                                         │
│  输出：ConflictType (NONE | SUPPLEMENT | SWITCH | CANCEL)               │
└─────────────────────────────────────────────────────────────────────────┘
```
